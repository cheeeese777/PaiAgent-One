# PaiAgent-One 用户使用手册

## 1. 环境准备

### 系统要求
- Java 17+
- Node.js 18+
- MySQL 8.0+
- Maven 3.8+

### 数据库配置

1. 启动 MySQL，创建数据库（或让应用自动创建）：
```sql
CREATE DATABASE paiagent DEFAULT CHARACTER SET utf8mb4;
```

2. 修改后端配置文件 `paiagent-backend/src/main/resources/application.yml`，根据你的环境调整：
```yaml
spring:
  datasource:
    url: jdbc:mysql://localhost:3306/paiagent?useUnicode=true&characterEncoding=utf-8&serverTimezone=Asia/Shanghai&createDatabaseIfNotExist=true
    username: root        # 改为你的数据库用户名
    password: 123456        # 改为你的数据库密码
```

## 2. 启动项目

### 启动后端
```bash
cd paiagent-backend
mvn spring-boot:run
```
启动成功后控制台会显示 `Started PaiAgentApplication`，服务运行在 `http://localhost:8080`。

Flyway 会在首次启动时自动执行数据库迁移：
- 创建 4 张表（users, node_definitions, workflows, workflow_executions）
- 插入默认管理员账号和节点定义数据

### 启动前端
```bash
cd paiagent-frontend
npm install       # 首次启动需要安装依赖
npm run dev
```
前端运行在 `http://localhost:5173`，已配置代理自动转发 API 请求到后端。

## 3. 登录系统

打开浏览器访问 `http://localhost:5173`，使用默认管理员账号登录：

- 用户名：`admin`

登录成功后自动跳转到工作流编辑器主界面。

### 4.1 顶部导航栏

| 元素 | 功能 |
|---|---|
| 工作流名称输入框 | 编辑当前工作流的名称 |
| **+ 新建** | 清空画布，创建一个新的空白工作流（含默认的输入和输出节点） |
| **加载** | 弹出对话框，从已保存的工作流列表中选择并加载 |
| **保存** | 将当前画布状态保存到数据库（首次保存为新建，之后为更新） |
| **调试** | 展开/收起底部调试面板 |
| 用户名 / 登出 | 显示当前登录用户，点击登出退出系统 |

### 4.2 左侧节点库

节点按分类展示，点击分类名可展开/折叠：

**大模型节点**（类型: LLM）
- DeepSeek - 深度求索大模型
- 通义千问 - 阿里云大模型
- AI Ping - AI Ping 大模型
- 智谱 - 智谱清言大模型

**工具节点**（类型: TOOL）
- 超拟人音频合成 - 文本转语音工具

### 4.3 中间画布区域

画布是工作流编辑的核心区域，支持以下操作：

- **拖拽添加节点**: 从左侧节点库拖拽节点到画布上
- **连线**: 从一个节点底部的连接点拖拽到另一个节点顶部的连接点
- **选中节点**: 点击节点，右侧显示该节点的配置面板
- **移动节点**: 按住节点拖拽移动位置
- **删除节点**: 选中节点后按 `Delete` 键删除
- **画布缩放**: 鼠标滚轮缩放，左下角有 +/- 按钮
- **画布拖拽**: 在空白区域按住鼠标拖拽移动画布
- **右下角小地图**: 显示全局缩略图，便于导航

### 4.4 右侧节点配置面板

点击画布上的节点后，右侧面板显示该节点的配置项：

**所有节点通用信息：**
- 节点 ID（只读）
- 节点类型（只读）

**LLM 节点特有配置：**
- 提供商（只读，由拖拽的节点决定）
- 模型选择（下拉菜单，如 qwen-turbo / qwen-plus）
- System Prompt（系统提示词文本框）
- Temperature（温度滑块，0-2）

**工具节点特有配置：**
- 工具类型（只读）
- 音色选择（男声/女声/解说员）

**输出节点特有配置：**
- 输出映射表：定义输出参数的名称和引用来源
  - 名称: 参数名（如 `output`）
  - 模式: 引用 / 固定值
  - 引用值: 选择上游节点的输出字段（如 `超拟人音频合成.audioUrl`）
- 回答内容配置: 使用 `{{参数名}}` 语法编写模板（如 `{{output}}`）

## 5. 操作流程

### 5.1 创建一个 AI 播客工作流

**步骤 1: 添加节点**
1. 从左侧"大模型节点"中拖拽"通义千问"到画布中部
2. 从左侧"工具节点"中拖拽"超拟人音频合成"到画布下方

**步骤 2: 连线**
1. 从"输入"节点底部的蓝色连接点 → 拖拽到"通义千问"节点顶部的连接点
2. 从"通义千问"底部 → 拖拽到"超拟人音频合成"顶部
3. 从"超拟人音频合成"底部 → 拖拽到"输出"节点顶部

最终连线: 输入 → 通义千问 → 超拟人音频合成 → 输出

**步骤 3: 配置 LLM 节点**
1. 点击"通义千问"节点
2. 在右侧面板选择模型（如 `qwen-turbo`）
3. 填写 System Prompt（如"你是一个专业的播客主播，请将用户输入的内容改编为生动的播客稿件"）
4. 调整 Temperature（建议 0.7-0.9）

**步骤 4: 配置输出节点**
1. 点击"输出"节点
2. 在输出映射中设置: 名称=`output`，模式=`引用`，值=`超拟人音频合成.audioUrl`
3. 回答内容配置填写: `{{output}}`

**步骤 5: 保存**
1. 在顶部输入工作流名称（如"AI播客生成器"）
2. 点击"保存"按钮

### 5.2 调试工作流

1. 点击顶部"调试"按钮，底部展开调试面板
2. 在左侧"输入内容"文本框中输入测试文字，例如：
   ```
   今天我们来聊一聊人工智能在医疗领域的最新应用
   ```
3. 点击"运行"按钮
4. 观察中间"执行进度"栏，各节点会依次显示执行状态
5. 执行完成后，右侧"执行结果"栏显示最终输出

> 注意：当前使用 Mock 模式，大模型返回模拟文本，语音合成返回模拟音频 URL。对接真实 API 后将返回实际内容。

### 5.3 加载已保存的工作流

1. 点击顶部"加载"按钮
2. 在弹出的对话框中选择之前保存的工作流
3. 画布自动恢复为保存时的状态

### 5.4 创建新工作流

1. 点击顶部"+ 新建"按钮
2. 画布重置为空白状态（仅包含默认的输入和输出节点）
3. 从头开始编排新的工作流

## 6. 节点类型说明

### 输入节点
- 每个工作流有且仅有一个输入节点
- 接收用户在调试面板中输入的文本
- 输出字段: `text`（用户输入的文本内容）

### LLM 大模型节点
- 将上游节点的文本输出作为 prompt 发送给大模型
- 可配置: 模型、系统提示词、温度
- 输出字段: `text`（模型生成的文本）、`tokenCount`（Token 数量）

### 工具节点（超拟人音频合成）
- 将上游节点的文本输出转换为语音
- 可配置: 音色、语速
- 输出字段: `audioUrl`（音频文件地址）、`duration`（时长）、`text`（原始文本）

### 输出节点
- 每个工作流有且仅有一个输出节点
- 通过输出映射收集上游节点的结果
- 通过模板语法 `{{参数名}}` 组装最终响应内容

## 7. 常见问题

**Q: 保存时提示失败怎么办？**
A: 确认后端服务正在运行，且数据库连接正常。检查浏览器控制台的网络请求错误信息。

**Q: 调试时提示"请先保存工作流后再调试"？**
A: 工作流必须先保存到数据库，才能通过后端引擎执行。点击"保存"按钮后再调试。

**Q: 拖拽节点到画布没有反应？**
A: 确保从节点库中按住节点拖拽，而不是单击。拖拽时需要移动到画布区域后松开鼠标。

**Q: 连线时无法连接两个节点？**
A: 连线方向为从源节点底部的输出连接点拖拽到目标节点顶部的输入连接点。输入节点只有输出连接点，输出节点只有输入连接点。

**Q: 如何删除一条连线？**
A: 点击选中连线后按 `Delete` 键删除。

**Q: 大模型返回的是模拟内容？**
A: 当前项目使用 Mock 模式运行。需要对接真实大模型 API（配置 API Key 并实现对应的 LLMProvider）后才能获得真实回复。
